## Q. What is the Compilation Process?

**Ans:**
The Compilation Process is the process of converting **human-readable source code** into **machine code (binary code)** so that the computer can understand and execute the program.

### Why do we need a Compiler?

* Computers can only understand **machine language (0s and 1s)**.
* Programmers write code in **high-level languages** like C++ which are easy for humans to understand.
* A **compiler** acts as a translator between the programmer and the computer.

### How does a Compiler help?

* Converts source code into machine code.
* Detects **syntax errors** in the program.
* Generates **warnings** for suspicious or unsafe code.
* Helps programmers identify and fix mistakes before execution.
* Improves the overall reliability of the program.

### Compilation Flow

```text
    C++ Source Code
       ↓
    Compiler
       ↓
    Machine Code
       ↓
    Executable File
       ↓
    Program Runs
```

### Key Points

* C++ code is written in a human-readable form.
* The compiler translates it into machine-readable code.
* Errors and warnings are displayed during compilation.
* A program can run only after successful compilation.

**Simply:**

> "A compiler is a translator that converts human-readable code into machine-readable instructions."
